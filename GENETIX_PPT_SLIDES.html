🔍 Problem Statement

Why Current AI Website Builders
Fall Short

Existing AI website builders like Lovable, Bolt.new, and v0.dev promise rapid website generation — but they suffer from critical limitations that prevent real-world adoption.

🔒

Single Framework Lock-in

Lovable & Bolt only support React. No option for Vue, Angular, Svelte, or plain HTML/CSS/JS. Developers are forced into one ecosystem regardless of project needs.

🖼️

No Real Image Integration

Competitors generate placeholder images or broken links. No automatic fetching of contextually relevant, high-quality stock photos from real APIs like Unsplash/Pexels.

🧠

No Persistent Memory

AI agents forget previous conversations. If you ask "change the hero section color," the agent doesn't remember the code it generated 2 messages ago. Context is lost.

💳

Limited Payment & Indian Market Support

No Indian payment gateways (Razorpay/Cashfree). No flexible credits system. Only expensive monthly subscriptions with rigid pricing — not suited for Indian developers.

🤖 AI / ML Component

Multi-Agent AI Pipeline

Genetix uses a production-grade multi-agent architecture powered by Large Language Models (LLMs) with intelligent tool use and persistent memory.

🧠 Model Used

OpenAI GPT-5.1 for high-quality code generation. GPT-5-mini as a cost-effective fallback. User can toggle Advanced Reasoning for complex projects.

Large Language Model (LLM)

🏗️ Agent Architecture

Inngest Agent Kit powers a multi-agent network: a Code Agent (generates & edits code), a Response Agent (summarizes output), and a Supervisor (orchestrates routing).

Multi-Agent System

🛠️ Agent Tools

createOrUpdateFiles — writes code to sandbox. terminal — runs commands (npm install, etc.). readFiles — reads existing code for context.

Tool-Augmented AI

💾 Memory & Context

Full conversation history (all messages + code fragments) persisted in PostgreSQL. Agent receives entire chat + latest code on every request.

Persistent Memory

🖼️ Image Intelligence

AI extracts semantic keywords from user prompt → queries Unsplash & Pexels APIs → injects high-res images as base64 into generated code automatically.

NLP + Image Pipeline

☁️ Deployment

AI agents run as Inngest serverless functions. Code executes in E2B cloud sandboxes (isolated Linux VMs). Stateless, scalable, production-ready.

Serverless + Edge

📊 Why LLM over Traditional ML?

Website generation requires understanding natural language intent, generating structured code across multiple frameworks, and contextually reasoning about design decisions. These tasks demand the generative and reasoning capabilities of Large Language Models — traditional ML (classification, regression) cannot produce structured multi-file code from free-form text. LLMs with tool-use enable a conversational, iterative development loop that mirrors how a human developer works.
